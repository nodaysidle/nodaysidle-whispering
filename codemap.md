# Repository Atlas: nodaysidle-whispering

## Responsibility
`nodaysidle-whispering` is a local-first desktop dictation utility that captures microphone audio, detects speech, transcribes it with Whisper.cpp via Rust bindings, and either types or pastes the result into the active application. It is intentionally privacy-first and on-device; it is not a cloud transcription service or a collaborative backend app.

## Tech Stack
- Frontend: React 19 + TypeScript + Vite + Tailwind CSS v4
- Backend: Rust
- Desktop/Mobile: Tauri 2 desktop app
- Database / Storage: local filesystem for Whisper model binaries in `models/`; no database layer found
- Key integrations: `whisper-rs`/Whisper.cpp, `cpal`, `webrtc-vad`, `global-hotkey`, `enigo`, clipboard/input injection via Rust

## System Entry Points
- `src/main.tsx` — React entry that mounts `App` and loads `style.css`
- `src/App.tsx` — frontend composition root; wires Tauri events, status state, controls, transcript view, and settings panel
- `src/tauri.ts` — typed frontend wrappers around Tauri `invoke()` commands
- `src-tauri/src/main.rs` — native binary entry; delegates to the library run function
- `src-tauri/src/lib.rs` — Tauri app composition root; owns app state, audio startup, thread lifecycle, shutdown, and command registration
- `README.md` — operator-facing overview, setup, and project structure summary

## Data & Control Flow
1. `src/main.tsx` mounts `App`, which loads initial dictation status and subscribes to Tauri events.
2. UI actions in `App.tsx` call wrappers in `src/tauri.ts`, which invoke Rust commands such as `load_model`, `start_recording`, `set_continuous_mode`, and `register_push_to_talk_hotkey`.
3. `src-tauri/src/main.rs` registers Tauri commands; `src-tauri/src/lib.rs` manages shared `AppState`, starts audio capture, and spawns the transcription pipeline threads.
4. `src-tauri/src/pipeline.rs` consumes audio frames, applies VAD, sends inference jobs to Whisper, then emits `dictation:update`, `dictation:hotkey`, and `dictation:error` events back to the frontend.
5. Final or partial transcription text is either displayed in the UI, copied/pasted, or injected into the active application depending on insertion mode.

## Integration Points
- **Database:** none found
- **Auth:** none found
- **External APIs:** none found; inference is local through Whisper bindings
- **Filesystem:** reads local model paths provided by the user; bundles frontend from `dist/`; uses macOS entitlements in `src-tauri/entitlements.plist`
- **Platform bridges:** Tauri `invoke()` commands and emitted events connect React to Rust; `global-hotkey` handles push-to-talk shortcuts; `enigo`/clipboard tooling inject text into the OS

## Constraints / Gotchas
- A Whisper model file must exist locally and be loaded before recording can start.
- The app is local-first and privacy-first; avoid introducing cloud dependencies as default behavior.
- Text injection and global hotkeys are OS-level features, so macOS permissions/entitlements can affect behavior.
- `dist/`, `node_modules/`, and `src-tauri/target/` are generated/build artifacts and should not be treated as source architecture.
- The frontend dev server is pinned to `127.0.0.1:1421`, and Tauri expects that exact dev URL during development.
